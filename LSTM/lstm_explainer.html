Two memory lanes

Every LSTM cell carries two streams of information across timesteps. Think of them as two kinds of memory your brain uses.

Cell state
Long-term memory — flows mostly unchanged, touched only by the gates
Hidden state
Short-term / working memory — the actual output passed to the next step
📖 Analogy: Reading a novel
Cell state is like remembering that the main character is an orphan — a fact established on page 3 that matters 400 pages later.

Hidden state is like your current understanding of the last paragraph you read — relevant right now, but replaced as you move on.

The four components

Each gate is a tiny neural network that outputs a value between 0 and 1 — a dimmer switch for information flow. 0 means "block everything", 1 means "let everything through".

🗑️
Forget Gate
What to erase from long-term memory
Looks at the current input and the previous hidden state, then decides how much of the cell state (long-term memory) to throw away. A value near 0 means "forget this"; near 1 means "keep it".
⚙ sigmoid activation
✏️
Input Gate
How much new info to store
Decides which new information is worth saving into long-term memory. It acts as a filter — even if candidate values are generated, this gate controls how much of them actually gets written.
⚙ sigmoid activation
💡
Candidate Values
Draft of what could be remembered
Creates a fresh draft of potential new memories using a tanh function (outputs -1 to 1). The input gate then filters this draft before writing any of it into the cell state.
⚙ tanh activation
📤
Output Gate
What to say out loud right now
Decides what part of the updated cell state should become the hidden state (working output). It squishes the cell state through tanh and filters it — producing the output for this timestep.
⚙ sigmoid + tanh

Step-by-step walkthrough

Here's what happens inside one LSTM cell at a single timestep. Click through each step.

STEP 01 / 05
Inputs arrive
At every timestep, the LSTM receives two things: the current input xₜ (the word, value, or feature at this moment) and the previous hidden state hₜ₋₁ (what the cell "said" last time). These two are combined and fed into all gates simultaneously.
Current input xₜ
+
Prev hidden hₜ₋₁
→
All gates
Example: processing the word "Paris" after reading "She moved to"
STEP 02 / 05
Forget gate decides what to erase
The forget gate looks at the inputs and produces a number between 0 and 1 for each piece of information in the cell state. Values close to 0 get erased; values close to 1 are preserved. This is how the LSTM "forgets" outdated context.
f = sigmoid(W·[hₜ₋₁, xₜ])
→
Cₜ₋₁ × f
If the subject changed from "he" to "she", the gender info in memory gets forgotten
STEP 03 / 05
Candidate values are proposed
A tanh layer creates a vector of candidate new memories — things that could potentially be stored. These are raw proposals; the input gate will decide how many of these to actually write. tanh outputs range from -1 to 1, encoding direction and magnitude.
C̃ = tanh(W·[hₜ₋₁, xₜ])
Draft: "The subject is now in Paris, in France, in Europe…"
STEP 04 / 05
Input gate writes to cell state
The input gate (sigmoid) decides how much of the candidate values to write. It multiplies with the candidates, then that result is added to the already-filtered old cell state. This produces the updated cell state Cₜ — the new long-term memory.
Cₜ₋₁ × f
+
i × C̃
=
New Cₜ
Old memory (partially erased) + new memory (partially written) = updated cell state
STEP 05 / 05
Output gate produces the hidden state
The output gate decides what to actually output from the cell state. It squishes the cell state through tanh (to get values between -1 and 1), then filters it with a sigmoid gate. The result is the new hidden state hₜ — both the output of this step and the "working memory" passed to the next step.
o = sigmoid(W·[hₜ₋₁, xₜ])
hₜ = o × tanh(Cₜ)
Output: the cell's summarized understanding, ready for the next word or the final prediction

Why not just RNN?

Regular RNNs have one fatal flaw: the vanishing gradient problem. LSTMs were designed specifically to fix it.

🔴 Vanilla RNN
✗ Gradients vanish over long sequences — it "forgets" early inputs
✗ Can't learn dependencies spanning more than ~10 steps
✗ Single hidden state = memory AND output mixed together
✗ No control over what to remember or forget
🟢 LSTM
✓ Cell state acts as a gradient highway — errors flow back hundreds of steps
✓ Learns long-range dependencies (e.g., matching "The cats… were")
✓ Separates long-term (cell) and short-term (hidden) memory
✓ Gates give explicit control over information flow
