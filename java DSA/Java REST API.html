Java REST API — HackerRank prep

Focus areas based on common assessment patterns

Making GET requests
HttpClient (Java 11+) Most common
Basic GET request
import java.net.http.*;
import java.net.URI;

HttpClient client = HttpClient.newHttpClient();

HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.example.com/users"))
    .header("Accept", "application/json")
    .GET()
    .build();

HttpResponse<String> response =
    client.send(request, HttpResponse.BodyHandlers.ofString());

int statusCode = response.statusCode();  // 200, 404, etc.
String body = response.body();           // raw JSON string
Tip: HackerRank gives you the API URL. You call it, parse the response body, then compute the answer.
Handling query parameters
Often Needed 
String url = "https://api.example.com/posts?userId=1&status=active";

HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create(url))
    .GET()
    .build();
Just build the URL string with params appended. No special library needed.
Checking status codes
if (response.statusCode() == 200) {
    String json = response.body();
    // parse and process
} else {
    System.out.println("Error: " + response.statusCode());
}
Always check status before parsing. 200 = OK, 404 = not found, 500 = server error.
Parsing JSON with org.json
Available Environment Easiest approach 
import org.json.JSONArray;
import org.json.JSONObject;

String jsonStr = response.body();

// Parse a JSON object
JSONObject obj = new JSONObject(jsonStr);
String name = obj.getString("name");
int age  = obj.getInt("age");
boolean active = obj.getBoolean("active");

// Parse a JSON array
JSONArray arr = new JSONArray(jsonStr);
for (int i = 0; i < arr.length(); i++) {
    JSONObject item = arr.getJSONObject(i);
    System.out.println(item.getString("title"));
}
Most HackerRank problems return a JSON array of objects. Loop through it with the pattern above.

Filter a list from API response
High Probability 
JSONArray arr = new JSONArray(response.body());
List<String> result = new ArrayList<>();

for (int i = 0; i < arr.length(); i++) {
    JSONObject item = arr.getJSONObject(i);
    
    // Filter: only active users in city "Delhi"
    if (item.getString("city").equals("Delhi")
        && item.getBoolean("active")) {
        result.add(item.getString("name"));
    }
}
System.out.println(result.size());
Sort by a field
List<JSONObject> list = new ArrayList<>();
for (int i = 0; i < arr.length(); i++)
    list.add(arr.getJSONObject(i));

// Sort by "salary" descending
list.sort((a, b) ->
    b.getInt("salary") - a.getInt("salary")
);

// Sort by string field
list.sort((a, b) ->
    a.getString("name").compareTo(b.getString("name"))
);
Find max/min/sum from API data
int max = Integer.MIN_VALUE;
int sum = 0;

for (int i = 0; i < arr.length(); i++) {
    int val = arr.getJSONObject(i).getInt("score");
    sum += val;
    if (val > max) max = val;
}
double avg = (double) sum / arr.length();
Convert JSONArray → List for Streams
Optional 
// First convert to List<JSONObject>
List<JSONObject> items = new ArrayList<>();
for (int i = 0; i < arr.length(); i++)
    items.add(arr.getJSONObject(i));

// Now use streams
long count = items.stream()
    .filter(o -> o.getInt("age") > 30)
    .count();

int total = items.stream()
    .mapToInt(o -> o.getInt("salary"))
    .sum();

Optional<JSONObject> top = items.stream()
    .max(Comparator.comparingInt(o -> o.getInt("score")));
Streams will feel natural. But the standard for-loop approach is totally fine on HackerRank.
Typical HackerRank task structure
Must know 
import java.net.http.*;
import java.net.URI;
import org.json.*;
import java.util.*;

public class Solution {
    public static void main(String[] args) throws Exception {
        // 1. Build client + request
        HttpClient client = HttpClient.newHttpClient();
        HttpRequest req = HttpRequest.newBuilder()
            .uri(URI.create("https://jsonmock.hackerrank.com/api/..."))
            .GET().build();

        // 2. Send and get body
        String body = client.send(req,
            HttpResponse.BodyHandlers.ofString()).body();

        // 3. Parse JSON
        JSONObject root = new JSONObject(body);
        JSONArray data = root.getJSONArray("data");

        // 4. Process (filter, sort, count, sum...)
        int answer = 0;
        for (int i = 0; i < data.length(); i++) {
            JSONObject item = data.getJSONObject(i);
            // your logic here
        }

        // 5. Print result
        System.out.println(answer);
    }
}
Assessments often use jsonmock.hackerrank.com as the fake API endpoint. The response usually has a "data" array and "total_pages" for pagination.
Pagination (multi-page APIs)
Frequently tested 
int page = 1;
int totalPages = 1;
List<JSONObject> all = new ArrayList<>();

do {
    String url = "https://jsonmock.hackerrank.com/api/movies"
                 + "?page=" + page;
    String body = client.send(
        HttpRequest.newBuilder().uri(URI.create(url)).GET().build(),
        HttpResponse.BodyHandlers.ofString()
    ).body();

    JSONObject root = new JSONObject(body);
    totalPages = root.getInt("total_pages");
    JSONArray data = root.getJSONArray("data");

    for (int i = 0; i < data.length(); i++)
        all.add(data.getJSONObject(i));

    page++;
} while (page <= totalPages);
Crucial: Always check "total_pages" in the first response and loop through all pages before processing.
