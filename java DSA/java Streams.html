Java Streams — API Assessment Guide

Core methods explained with realistic REST API data manipulation examples

intermediate keeps matching items

Takes a predicate (true/false condition). Keeps only items where the condition is true. Returns a new stream — does not modify the original list.

List of matches → .filter(goals > 2) → only high-scoring 
// Data: list of JSONObjects from football_matches API
List<JSONObject> matches = /* from API */;

// Keep only matches where team1 scored more than 2 goals
List<JSONObject> highScoring = matches.stream()
    .filter(m -> m.getInt("team1goals") > 2)
    .collect(Collectors.toList());

// Multiple conditions — chain with &&
List<JSONObject> draws = matches.stream()
    .filter(m ->
        m.getInt("team1goals") ==
        m.getInt("team2goals"))
    .collect(Collectors.toList());
output
highScoring → only matches where team1goals > 2
draws → only matches where scores were equal
intermediate transforms each item

Applies a function to every element and returns a stream of the transformed results. Use it to extract a field, convert a type, or reshape an object.

List<JSONObject> → .map(get "winner") → List<String> 
// Extract winner names from competitions API response
List<String> winners = competitions.stream()
    .map(c -> c.getString("winner"))
    .collect(Collectors.toList());

// mapToInt — extracts an int field (enables sum/avg/max)
int[] goalCounts = matches.stream()
    .mapToInt(m -> m.getInt("team1goals"))
    .toArray();

// map to compute a new value per item
List<Integer> totalGoals = matches.stream()
    .map(m ->
        m.getInt("team1goals") +
        m.getInt("team2goals"))
    .collect(Collectors.toList());
output
winners → ["Chelsea", "Barcelona", "Bayern"]
totalGoals → [3, 2, 5, 1, 4, ...]
intermediate terminates the stream

Collects the stream back into a concrete collection. It's a terminal operation — nothing runs until you call collect(). Three variants you need to know:

import java.util.stream.Collectors;

// 1. toList() — most common, collect to ArrayList
List<String> names = matches.stream()
    .map(m -> m.getString("team1"))
    .collect(Collectors.toList());

// 2. toSet() — removes duplicates automatically
Set<String> uniqueTeams = matches.stream()
    .map(m -> m.getString("team1"))
    .collect(Collectors.toSet());

// 3. joining() — concatenates strings
String csv = matches.stream()
    .map(m -> m.getString("team1"))
    .collect(Collectors.joining(", "));

// 4. counting() — count matching items
long count = matches.stream()
    .filter(m -> m.getInt("team1goals") > 2)
    .collect(Collectors.counting());

// 5. summingInt() — sum an int field
int totalGoals = matches.stream()
    .collect(Collectors.summingInt(
        m -> m.getInt("team1goals")));
output examples
uniqueTeams → [Barcelona, Chelsea, Bayern] (no duplicates)
csv → "Chelsea, Barcelona, Bayern, Chelsea"
count → 7
totalGoals → 34
intermediate returns Optional

max() and min() return an Optional<T> — you must call .get() or .orElse() to unwrap the value. Pass a Comparator to tell it what to compare by.

// Find match with most team1 goals
Optional<JSONObject> topMatch = matches.stream()
    .max(Comparator.comparingInt(
        m -> m.getInt("team1goals")));

// Safely unwrap with .get()
if (topMatch.isPresent()) {
    JSONObject m = topMatch.get();
    System.out.println(m.getString("team1"));
}

// Or use .orElse() to provide a default
JSONObject best = matches.stream()
    .max(Comparator.comparingInt(
        m -> m.getInt("team1goals")))
    .orElse(null);

// mapToInt + max() — simpler when you just need the number
int maxGoals = matches.stream()
    .mapToInt(m -> m.getInt("team1goals"))
    .max()
    .orElse(0);

// min() — same pattern, opposite result
int minGoals = matches.stream()
    .mapToInt(m -> m.getInt("team1goals"))
    .min()
    .orElse(0);
output
maxGoals → 5
minGoals → 0
best → JSONObject of the highest-scoring match
advanced returns Map

groupingBy() splits a list into groups based on a key. Returns a Map<Key, List<T>>. Very useful for counting per category or summing per group.

import java.util.stream.Collectors;
import java.util.*;

// Group matches by team1 name
Map<String, List<JSONObject>> byTeam = matches.stream()
    .collect(Collectors.groupingBy(
        m -> m.getString("team1")));
// { "Chelsea": [...], "Barcelona": [...], ... }

// Count matches per team
Map<String, Long> matchCount = matches.stream()
    .collect(Collectors.groupingBy(
        m -> m.getString("team1"),
        Collectors.counting()));
// { "Chelsea": 6, "Barcelona": 8, ... }

// Sum goals per team (most common in IBM tests)
Map<String, Integer> goalsByTeam = matches.stream()
    .collect(Collectors.groupingBy(
        m -> m.getString("team1"),
        Collectors.summingInt(
            m -> m.getInt("team1goals"))));
// { "Chelsea": 14, "Barcelona": 22, ... }

// Find team with most goals using max on the map
String topScorer = goalsByTeam.entrySet().stream()
    .max(Map.Entry.comparingByValue())
    .get().getKey();
output
matchCount → {Chelsea=6, Barcelona=8, Bayern=5}
goalsByTeam → {Chelsea=14, Barcelona=22, Bayern=11}
topScorer → "Barcelona"
real exam pattern

In assessment tests you almost always chain multiple methods. Here's a realistic multi-step problem: from paginated API data, find the team with the most wins (where a win = more goals than opponent).

raw matches → .filter(team1 won) → .map(team1 name) → groupingBy + count → team with max wins 
// Step 1: collect all matches from all pages first
List<JSONObject> all = /* paginated fetch */;

// Step 2: chain filter → map → collect → max
String mostWins = all.stream()

    // keep only matches where team1 won
    .filter(m ->
        m.getInt("team1goals") >
        m.getInt("team2goals"))

    // extract team1 name
    .map(m -> m.getString("team1"))

    // group by name and count wins
    .collect(Collectors.groupingBy(
        t -> t,
        Collectors.counting()))

    // get team with highest count
    .entrySet().stream()
    .max(Map.Entry.comparingByValue())
    .get()
    .getKey();

System.out.println(mostWins);
output
mostWins → "Barcelona"

The mental model: think of a stream as an assembly line. Raw data comes in, each method transforms or filters it, and collect() / max() / count() terminates the line and gives you the final result.
